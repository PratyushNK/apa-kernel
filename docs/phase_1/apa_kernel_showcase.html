System Architecture
Four Decoupled Subsystems

Each subsystem communicates only through shared files on disk — no direct coupling. The simulator emits events. The kernel observes, reasons, and adapts.

⚡
Transaction Simulator
Emits realistic payment events to JSONL stream. Poisson arrivals, Markov gateway regimes, LogNormal latency.
📊
Metrics Aggregator
Reads event stream incrementally. Computes 11 windowed metrics. Tracks last-healthy baseline for delta.
🧠
Kernel Engine
Monitors aggregator. Detects invariant breaches. Orchestrates adaptation with cooldown control.
🔄
Adaptation Loop
Two-stage LLM reasoning. Proposes policy vector θ. Verifies invariants. Deploys and observes outcome.
simulator/runner.py → writes to data/streams/events.jsonl  ·  kernel/engine/runner.py → reads from data/streams/events.jsonl  ·  adaptation writes to data/policies/policy.json  ·  simulator reloads policy.json on every routing decision

Formal Safety
Invariants Preserved Throughout

Every proposed policy vector was validated against the full invariant set before deployment. No invariant was violated at any point during the adaptation process.

I1
Single Settlement
No transaction processed more than once. Verified via attempt ID uniqueness.
I2
Retry Bound
max_retry proposed as 2 — within [1,5] bound. Enforced by InvariantVerifier.
I3
Terminal Absorption
No AttemptExecution emitted after terminal state. Enforced in TransactionEngine.
I4
Single Active Attempt
No concurrent attempts per transaction. State machine enforces sequential execution.
I6
Circuit Respect
G1 circuit OPEN — routing correctly excluded G1 from selection via is_up() check.
I7
SLA Terminal
Transactions exceeding SLA deadline marked FAILED. No further attempts allowed.
